# Battle Modal Input Gate Design

## Problem

After an effect dialog or field interaction closes, the first click on a hand card can occasionally be ignored and the second click works. This points to a stale input suppression state rather than a card rule issue.

The current battle UI has several independent input guards:

- Dialog choices call `_mark_modal_input_consumed()`, which arms a short modal slot suppression window and marks the viewport input as handled.
- Hand cards emit drag-scroll input before their release click path. The hand drag coordinator can leave `_hand_drag_suppress_click_until_msec` active after a drag.
- Card search galleries have a separate drag capture and click suppression window.
- Effect dialogs, field slot prompts, field assignments, and card galleries each close through different controller methods.

This makes it possible for one path to close a modal while leaving stale hand drag or gallery drag state behind. The next hand-card tap then reaches `show_hand_card_detail()` while `_is_hand_drag_click_suppressed()` is still true, so it returns without visible feedback.

## Goal

Make modal completion reset stale transient input capture in one place, so the first intentional hand-card click after a popup closes is accepted.

## Non-Goals

- Replace Godot input dispatch globally.
- Remove drag-to-scroll behavior from hand or card-search galleries.
- Change effect selection semantics.

## Design

Introduce a scene-level helper:

```gdscript
_finish_modal_input_interaction(source: String, slot_suppression_mode: String = "arm")
```

This helper is the single exit point for modal input cleanup. It must:

- Cancel active card-gallery drag capture.
- Clear hand drag active and hand click suppression.
- Cancel slot long-press capture.
- Arm, clear, or preserve the short slot-input suppression used to prevent click-through on field slots.
- Mark the viewport event as handled.
- Emit a runtime log entry with the source and whether slot suppression was armed.

Dialog and field interaction controllers should call this helper instead of calling `_mark_modal_input_consumed()` plus ad hoc drag cleanup. The old methods remain as compatibility wrappers. The supported slot suppression modes are `arm`, `clear`, and `preserve`.

## Phase 1 Implementation

Keep the current time-window slot suppression, because several tests and portrait click-through fixes already rely on it for board slots, discard HUD, lost-zone HUD, and portrait bench fallback. Narrow the risk by clearing hand/card-gallery suppression whenever a modal interaction finishes.

Touch points:

- `BattleSceneDialogInteractionReviewRuntime.gd`
  - Add `_finish_modal_input_interaction()`.
  - Rework `_mark_modal_input_consumed()` and `_mark_modal_input_consumed_without_slot_suppression()` as wrappers.
- `BattleDialogController.gd`
  - Route modal completion through the new helper.
  - Keep `_hide_dialog_overlay()` focused on visual hiding.
- `BattleInteractionController.gd`
  - Route field slot, field assignment, and counter prompts through the new helper.
- `BattleEffectInteractionController.gd`
  - Use the helper when effect interaction state resets.
- `BattleDragScrollCoordinator.gd`
  - Expose a stronger reset that clears both hand and gallery transient capture.

## Future Phase

Replace the remaining time-window click-through guard with event-sequence tokens. That would suppress only events generated by the same pointer sequence that closed the modal, instead of suppressing any click that happens within a fixed number of milliseconds. This is more precise, but it needs broader pointer identity handling across mouse and touch input.

## Verification

Add UI regression tests:

- After dialog confirm, a stale hand-drag suppression is cleared and the next hand Item tap opens its Use/Cancel detail immediately.
- Field assignment source clicks that intentionally skip slot suppression still clear stale hand-drag suppression.
- Existing click-through suppression tests for slots, discard, lost-zone, and portrait bench fallback continue to pass.
- Hand/card-gallery drag tests continue to pass.
